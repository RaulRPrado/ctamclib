Data Model¶

Modules in the data_model sections provide functionality for reading, writing, and validation of data. Data products ingested or produced by simtools generally follows the CTA data model.

metadata_model¶

Definition of metadata model for input to and output of simtools. Follows CTA top-level data model definition. Metadata is broken into hierarchical categories in this module.

Metadata definitions require the three following fields to be defined:

  • type: data type

  • required: boolean if this field is required as input(!)

  • default: default value

These definitions are used for:

  • data products submitted to SimPipe (‘input’)

  • data products generated by SimPipe (‘output’)
